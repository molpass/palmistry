# 손금으로 보는 운세: 시점에 강인한 머신 손금 분석

> 이 저장소는 molpass가 포크한 사본입니다. 영어 원문은 [README.en.md](./README.en.md)를 참고하세요.

## 요약
이 *손금 주요선 검출 소프트웨어*는 아래 4단계로 구현되어 있습니다. 핵심 과제는 **촬영 방향**과 **조명**에 관계없이 손바닥의 주요선을 읽어내는 것이었습니다:
1) 기울어진 손바닥 이미지를 워핑(warping)
2) 손바닥의 주요선 검출
3) 선 분류
4) 각 선의 길이 측정

손바닥 이미지 정렬(rectification)에는 MediaPipe로 관심점(interest points)을 추출하고 그 점들로 워핑을 구현했습니다. 주요선 검출을 위해서는 딥러닝 모델을 구축하고 손바닥 이미지 데이터셋으로 학습시켰습니다. 선 분류에는 K-means 클러스터링을 사용해 각 픽셀을 특정 선에 할당했습니다. 길이 측정에는 MediaPipe로 얻은 랜드마크를 이용해 각 주요선에 대한 임계값(threshold)을 설정했습니다.

## 실행 환경
코드는 Python 3.7.6 기준으로 작성되었습니다. 실행에 필요한 요구사항은 다음과 같습니다:
- torch
- torchvision
- scikit-image
- opencv-python
- pillow-heif
- mediapipe

요구사항을 설치하려면 `pip install -r ./code/requirements.txt` 를 실행하세요.

## 실행
1. 코드를 실행하기 전에, **입력용 손바닥 이미지(.heic 또는 .jpg)** 를 `./code/inputs` 디렉터리에 준비해야 합니다. 네 개의 샘플 입력을 제공합니다.
2. 아래 명령으로 `read_palm.py` 를 실행합니다. 실행 후 결과 파일은 `./code/results` 디렉터리에 저장됩니다.
```bash
> python ./code/read_palm.py --input [filename].[jpg, heic]
```

## 선분 검출 구현
업데이트: 22.12.03 21:57
- 가정
  - 선이 이미지의 테두리까지 가는 경우가 없음 (이 경우 scikit의 skeletonize가 종종 안 됨. skeletonize 되더라도 그룹화 알고리즘을 약간 수정해야 함)
  - 선들이 교차하는 점은 최대 하나 (테스트 케이스에 따랐음. 약간의 추가 구현으로 처리 가능)

- 선 그룹화
  - 반환값: 선들의 리스트이며, 각 선 또한 픽셀들의 리스트입니다.
  ```
  예시: [ [[1, 2], [2, 3]], [[10, 11], [11, 11]] ]
  ```

  - 구현 설명
    1. 전체 픽셀에 대해 둘레 8픽셀 중 0이 아닌 값을 센다
    2. 센 결과는 0: 선 위에 없음, 1: 선의 끝, 2: 선의 중간, 3: 선의 교차점으로 구분된다
    3. 선의 끝인 픽셀에서 시작해 주변 8픽셀을 탐색하고, 값이 0이 아니며 방문하지 않은 픽셀을 따라간다
    4. 진행하다 보면 값이 1이나 3인 픽셀에 도달한다
    5. 1인 픽셀이면 선을 하나 찾은 것이므로 저장하고 역방향 탐색을 막기 위해 반복문에서 제외한다. 3인 픽셀은 선을 따로 저장해 두었다가 추후 처리한다
    6. 3으로 끝난 선들끼리 이을 수 있는지 확인한다: 시작점·끝점 차이를 확인해 방향이 반대인 모든 조합을 이어 선에 저장한다
    7. 저장한 선들을 반환한다

## 이슈
  - skeletonize가 붙어 있지 않던 선을 붙이는 경우가 있음 (1건, 선 하나가 약간 길게 나옴) → 추가 테스트 필요
  - 끊어진 선 처리가 애매함: 현재는 무시하고 진행한 상태. 그룹화된 선들의 기울기(gradient)를 계산하면 처리할 수 있으나, 잘못하면 엉뚱한 선들끼리 이어질 수 있음. 이런 경우는 숨기는 편이 좋아 보임.
